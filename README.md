# LibrarySystemClient — Blazor-клиент для CoreWCF-сервиса

## Описание

Веб-приложение на Blazor Server для взаимодействия с CoreWCF-сервисом управления библиотекой (КТ-4). Поддерживает два транспорта (HTTP/TCP), аутентификацию, ролевую авторизацию, полный CRUD, обработку ошибок и unit-тесты.

## Способы взаимодействия с сервисом

| Транспорт | Адрес | Описание |
|-----------|-------|----------|
| HTTP | `http://localhost:5000/LibraryService.svc` | BasicHttpBinding (SOAP/HTTP) |
| TCP | `net.tcp://localhost:8090/LibraryService.svc` | NetTcpBinding (бинарный протокол) |

Переключение транспорта доступно на каждой странице через чекбокс **TCP**.

## Обработка ошибок (FaultException)

Все ошибки от CoreWCF-сервиса обрабатываются централизованно в `ErrorHandler`:

| Тип ошибки | Исключение | Отображение |
|------------|-----------|-------------|
| Доступ запрещён | `FaultException("Access denied...")` | «Доступ запрещён. Недостаточно прав» |
| Неверные данные | `FaultException("Invalid credentials")` | «Неверное имя пользователя или пароль» |
| Не найдено | `FaultException("...not found")` | «Запрашиваемый ресурс не найден» |
| Токен истёк | `FaultException("...expired")` | «Срок действия токена истёк» |
| Нет экземпляров | `FaultException("No available copies")` | «Нет доступных экземпляров книги» |
| Нет соединения | `CommunicationException` | «Ошибка соединения с сервером» |
| Таймаут | `TimeoutException` | «Превышено время ожидания» |

## Как запустить

```bash
# 1. Запустить CoreWCF-сервис (КТ-4)
cd ../peresdacha-kt4
dotnet run --project src/LibrarySystem.API

# 2. В другом терминале — запустить Blazor-клиент
cd ../peresdacha-kt5
dotnet run --project src/LibrarySystem.BlazorClient

# 3. Открыть в браузере
# http://localhost:5100

# Тесты
dotnet test
```
